## 1. 产品概述

个人播客订阅与收听管理APP，围绕用户日常播客消费场景设计，提供从发现、订阅、收听到回顾的完整闭环体验。目标用户为播客爱好者与内容创作者，解决播客管理分散、收听体验单一、缺乏个性化推荐和收听数据洞察的痛点。

- 核心价值：一站式播客消费管理，智能推荐与数据驱动收听洞察
- 目标市场：中文播客听众群体，兼顾内容创作者的节目上传与管理需求

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 邮箱注册 | 搜索/订阅播客、收听播放、笔记评分、查看报告 |
| 管理员 | 管理员分配 | 审核上传节目、管理分类标签、用户管理 |

### 2.2 功能模块

1. **发现页**：播客搜索、分类浏览、每周智能推荐
2. **播客详情页**：播客信息展示、剧集列表、订阅操作
3. **播放器页**：音频播放控制、倍速/定时/跳过静音、笔记与评分
4. **我的订阅页**：已订阅播客列表、最新剧集自动下载
5. **收听报告页**：每周收听报告、收听总时长/最爱节目/连续天数、PDF导出
6. **上传播客页**：节目上传表单（含校验）
7. **管理员审核页**：节目审核、分类标签管理

## 3. 核心流程

### 3.1 用户订阅与收听流程
用户在发现页搜索或浏览播客 → 进入播客详情页查看信息和剧集 → 点击订阅 → 系统自动下载最新集 → 在播放器中收听，可选择倍速/定时/跳过静音 → 收听中或收听后添加笔记和评分 → 系统记录收听历史

### 3.2 智能推荐流程
系统每周根据用户收听历史和偏好标签 → 计算相似度匹配新播客 → 生成推荐列表 → 在发现页展示推荐卡片 → 推送通知提醒用户查看

### 3.3 收听报告流程
每周一系统自动汇总上周收听数据 → 计算总时长、最爱节目、连续天数 → 生成可视化报告 → 用户可导出为PDF

### 3.4 管理员审核流程
用户上传播客节目（校验通过后进入待审核）→ 管理员在审核页查看 → 通过则上架到发现页，拒绝则通知用户原因

## 4. 用户界面设计

### 4.1 设计风格

- **整体风格**：暗夜暖调 — 深色背景搭配琥珀金色调，营造深夜电台般的沉浸聆听氛围
- **主色调**：深炭黑 (#1A1A2E) 为底，琥珀金 (#E8A838) 为强调色，暖灰 (#F0EDE8) 为文字
- **次色调**：深靛蓝 (#16213E) 用于卡片背景，柔白 (#F5F0EB) 用于浅色元素
- **按钮风格**：圆角微凸3D按钮，悬浮时有柔光投影，点击有下沉效果
- **字体**：标题使用 Noto Serif SC 衬线字体（优雅文艺感），正文使用 Noto Sans SC（清晰可读）
- **布局风格**：左侧导航栏 + 右侧内容区，卡片式内容展示，圆角卡片带微妙阴影
- **图标风格**：线性图标，使用 Lucide React 图标库，琥珀金色调
- **动效**：页面切换淡入淡出，卡片悬浮微浮起，播放器波形动画

### 4.3 响应式设计

- 桌面优先设计，最小宽度1024px
- 平板端（768-1024px）：导航栏折叠为汉堡菜单，卡片由三列变两列
- 移动端（<768px）：底部Tab导航，单列卡片，播放器全屏化
